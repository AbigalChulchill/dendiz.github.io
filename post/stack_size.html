Stack size in programs

2021-07-26

I've been thinking about why there is a limit on the function call stack. Why can't programs just recurse until there is no memory left on the host? It turns out that there are some architectural implications of having a large stack and it's also an error control mechanism.

Each thread gets it's own stack. But the memory layout for these stacks are contiguous. So it would not be possible to grow one of these stacks without affecting the others.
